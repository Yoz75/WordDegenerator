# WordDegenerator — краткая справка для пользователей

Коротко
-------
WordDegenerator принимает ваш текст, разбивает его на части (токены) разными способами и затем собирает новый текст из этих частей. Это позволяет получать похожие, но отличающиеся варианты исходного текста — от аккуратных и читаемых до более случайных и экспериментальных.

Как пользоваться
----------------
1. Выберите токенайзер — способ разбиения текста на части.
2. Настройте параметры токенайзера (чем мельче части, тем более детально будет «перемешан» текст).
3. Выберите генератор и его параметры — сколько частей сгенерировать и насколько часто выбирать случайный фрагмент.
4. Нажмите «Выполнить» — получите сгенерированный текст.

Токенайзеры (способы разбиения)
-------------------------------
1) BlockTokenizer — разбиение на блоки фиксированной длины
- Что делает: делит текст на кусочки одинаковой длины (например, по 3 символа) и работает с ними как с «словами».
- Параметры:
  - `BlockSize` — длина блока. Меньше значение = более мелкие части и более смешанные результаты. Рекомендуется: 2–4; по умолчанию: 3.
  - `FillChar` — символ, которым дополняется конец текста, если размер текста не делится на `BlockSize`. Обычно оставляют пробел.

2) SeparatorTokenizer — разбиение по разделителю (обычно пробел)
- Что делает: разбивает текст по символу-разделителю (например, пробел) и считает каждую полученную часть за элемент для дальнейшей генерации.
- Параметры:
  - `Separator` — символ, по которому делится текст (пробел, запятая и т.д.). Если хотите работать со словами, используйте пробел.

3) RandomBlockTokenizer — разбиение на блоки разной длины
- Что делает: похож на BlockTokenizer, но длина каждого блока выбирается случайно в указанном диапазоне. Даёт более разнообразные и непредсказуемые результаты.
- Параметры:
  - `Min` — минимальная длина блока. Рекомендуется ≥1; по умолчанию: 1.
  - `Max` — максимальная длина блока. Чем больше, тем крупнее фрагменты; по умолчанию: 3.
  - `FillChar` — символ дополнения в конце текста, если нужно.
  - Совет: держите `Min` меньше или равно `Max`.

4) SmartTokenizer — объединяет часто встречающиеся пары символов/фрагментов
- Что делает: автоматически находит часто повторяющиеся соседние фрагменты и объединяет их в более длинные элементы. Подходит, если хотите, чтобы наиболее частые сочетания оставались целыми.
- Параметры:
  - `IterationsCount` — сколько раз выполнять поиск и объединение. Больше итераций = более крупные и устойчивые фрагменты. Для средних и больних текстов рекомендуется 1000 и более.
  - `MaxTokenLength` — максимально допустимая длина объединённого фрагмента; помогает избежать слишком длинных элементов.
  - `TopK` — насколько широкий набор популярных пар рассматривать перед выбором для объединения. Меньше значение = более предсказуемый результат; больше = более вариативный.

Генераторы (как собирается новый текст)
--------------------------------------
1) DefaultTextGenerator — стандартный генератор
- Что делает: берет фрагменты (из токенайзера) и собирает из них последовательность нужной длины. Иногда выбирает следующий фрагмент по «логике» связи, иногда случайно — чтобы добавить разнообразие.
- Параметры:
  - `OutputSize` — сколько фрагментов сгенерировать. Чем больше — тем длиннее результат. По умолчанию: 50.
  - `TopK` — при выборе «следующего» фрагмента учитывается несколько наиболее подходящих вариантов; `TopK` задаёт, сколько вариантов рассматривать. Меньше = более плавный, предсказуемый текст; больше = больше вариативности. По умолчанию: 5.
  - `RandomTokenChance` — вероятность (от 0 до 1) выбрать случайный фрагмент вместо «логического» продолжения. 0 = без случайности, 1 = всегда случайно. Рекомендуется: 0–0.2 для небольших вариаций.
  - `UseSeparator` — если включено, в вывод добавляется видимый разделитель между фрагментами (полезно для отладки).

Рекомендации по настройке
-------------------------
- Для читаемых результатов: `SeparatorTokenizer` (по словам) или `BlockTokenizer` с `BlockSize` 2–4; в генераторе `RandomTokenChance` около 0.05–0.2.
- Для более экспериментальных/шумных результатов: `RandomBlockTokenizer` с широким диапазоном `Min`–`Max`, или `SmartTokenizer` с большим `IterationsCount`; в генераторе увеличьте `RandomTokenChance`.
